# Проблематика

- Розрізнені канали комунікації (Email, месенджери, WebSocket) потребують єдиного входу та узгоджених SLA.
- Сервіси, що генерують події, не повинні блокуватися очікуванням відповіді на відправку нотифікацій.
- Необхідна стійкість до збоїв: відновлення після падіння окремих worker-ів, повторна доставка та ізоляція невдалих повідомлень.
- Контроль стану бізнес-процесу (Saga) і прозорий моніторинг для операційних команд.
- Швидкий старт локальної розробки та демонстрації — один `docker-compose up`.
